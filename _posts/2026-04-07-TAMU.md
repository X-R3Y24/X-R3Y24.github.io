Well as the title suggests - abnormal (or anomalous) curve are linearly broken by Smart's Attack. All we need to do in this challenge is check if order of E == p and use P.log(G). That's it. So let's talk about what is Smart's attack and why does it work for such curves. 

Smart's attack uses a trick that transfers ECDLP into a finite field where the DLP problem is much easier than the original ECDLP. Smart's attack substitutes the original elliptic curve group with a basic ($\mathbb{F}_p$, +) group which results in switching a hard ECC addition into a basic addition. It then solves it linearly.

**Why is $\\#E = p$ the key condition?**

Smart's attack is based on *lifting* points from $E(\mathbb{F}_p)$ to $E(\mathbb{Q}_p)$ — the p-adic numbers.
Hensel's lemma guarantees that every point $P \in E(\mathbb{F}_p)$ has a unique lift $P̃ \in E(\mathbb{Z}/p^2\mathbb{Z})$.

The critical observation: when #E = p, the reduction map $E(\mathbb{Z}/p^2\mathbb{Z}) → E(\mathbb{F}_p)$ has a kernel
isomorphic to ($\mathbb{Z}/p\mathbb{Z}$, +). This means that for any lifted point P̃:

$$p \cdot \tilde{P} \equiv 0 \pmod{p^2} \quad \text{but} \quad p \cdot \tilde{P} \not\equiv 0 \pmod{p}$$

The formal group logarithm of E then allows us to compute:

$$k = \frac{(p \cdot \tilde{P})_x}{(p \cdot \tilde{G})_x} \pmod{p}$$

where $(·)_x$ denotes the x-coordinate divided by p — and this is exactly the discrete
logarithm k such that P = k * G. The hard group law on E($F_p$) has been replaced by
a simple division in $\mathbb{Z}/p\mathbb{Z}$.

**Why doesn't it work for $\\#E ≠ p$?**

When $\\#E ≠ p$, the kernel of the reduction map no longer aligns with ($\mathbb{Z}/p\mathbb{Z}$, +). The lift
still exists, but computing p · P̃ no longer yields a value linearly related to the scalar
k — the formal logarithm loses its foothold, and the attack breaks down.

That's more of our standard CTF problem. Here we are given a RSA + DLP encryption scheme and we need to find a vulnerability. It is nicely hidden here: s = randint(1, 1 << 100). It tells us that s which is our secret exponent in DLP is a random int between 1 and $2^{100}$. The problem is that $2^{101} \mid p - 1$. Since $2^{100} < 2^{101}$ we are entering the Pohlig-Hellman territory.

Since $p - 1$ has $2^{101}$ as a factor, there exists a subgroup of order $2^{101}$ in $\mathbb{F}_p^*$.
By projecting $g$ and $A$ into this subgroup (via exponentiating by $\frac{p-1}{2^{101}}$), we reduce
the original DLP to a DLP inside a group of order $2^{101}$. Since $s < 2^{100} < 2^{101}$,
the secret exponent fits entirely within this subgroup — no information is lost in the projection.
Pohlig-Hellman then solves the DLP in the subgroup by decomposing it into a sequence of DLPs
in groups of order 2, each of which is trivial. The results are combined via CRT to recover $s$.

```python
h = power_mod(g, (p-1) // 2^101, p)
a = power_mod(A, (p-1) // 2^101, p)
s = discrete_log(a, Mod(h, p), ord=2^101)
```
After recovering s using the Pohlig-Hellman we then use it to unmask d:
```python
print(s)
```

Now that we have d, we can factor n. Knowing d and e, we compute k = d * e - 1
which is a multiple of $\varphi(n)$. We then repeatedly halve k and exponentiate a
random base g until we find x such that $x^2 \equiv 1 \pmod{n}$ but $x \not\equiv \pm 1 \pmod{n}$.
At that point gcd(x-1, n) gives us a non-trivial factor of n.

```python
k = int(d * e - 1)
from random import randrange
while True:
    g = randrange(2, int(n))
    t = k
    while t % 2 == 0:
        t //= 2
    x = pow(g, t, int(n))
    while x != 1:
        y = pow(x, 2, int(n))
        if y == 1:
            p = gcd(x - 1, int(n))
            if 1 < p < int(n):
                q1 = p
                q2 = int(n) // p
                break
        x = y
    else:
        continue
    break
```

The last step is Rabin decryption. The flag was encrypted as $c = m^2 \bmod n$
(not standard RSA — note the exponent 2 in the script). We recover m by computing
square roots mod q1 and q2 separately, then combining all four combinations via CRT
and checking which candidate contains the flag prefix.

```python
r1 = Mod(c, q1).sqrt()
r2 = Mod(c, q2).sqrt()

for s1 in [r1, -r1]:
    for s2 in [r2, -r2]:
        x = crt([int(s1), int(s2)], [int(q1), int(q2)])
        candidate = long_to_bytes(int(x))
        if b'gigem' in candidate:
            print(candidate.decode())
```

### **Challenge 3: Random Password**

|------|---------|

| Solve method     | Timing attack on pruning tree using precomputed table with remaining 'time' |
| What was provided | a server.py file and a solver-template.py file             |
| Difficulty | Medium-hard |

**server.py:**

```python
from functools import partial
import re
import random

random.seed(121728)

def random_sleep(timeout: float) -> None:
  time_elapsed = 0
  while time_elapsed < timeout:
    time_elapsed += random.random()

handle_zero = partial(random_sleep, 5)
handle_one = partial(random_sleep, 17)

def verify(password: str, correct_value: float) -> bool:
  if len(password) != 64: return False

  bit_str = bin(int(password, 16))[2:].zfill(256)

  for bit in bit_str:
    match bit:
      case '0':
        handle_zero()
      case '1':
        handle_one()

  return random.random() == correct_value

password = input('Enter the password in hex: ')
if not re.match('^[0-9a-f]+$', password):
  print('Invalid nonhex input')
  exit()

if verify(password, 0.9992610559813815):
  with open('flag.txt', 'r') as outfile:
    print("Here's the flag", outfile.readline().strip())
    exit()
else:
  print('Incorrect password')
  exit()
```

The last challenge of my TAMU triplets and the one that was definitely the hardest for me, but made me proud. This challenge is designed to simulate a timing attack on a authentication mechanism where you need to input a correct random password. Each bit has its own 'time'. Let's call it cycles. Zeros need 5 cycles to compute while ones need 17 cycles. First of all let's check how many cycles overall we have:

It tells us that we have 5720 cycles before reaching the target. How do we know that this is what we are looking for? Check this:

The server will return flag only if our password corresponds to this value. That's why I defined target as 0.9992610559813815. After that we append all the states into an array to construct two arrays named fives and seventeens. They are defined as follows: fives[i] = the maximum number of bits we can still place starting from position i in the states array,
assuming all remaining bits are zeros (each costing 5 cycles). Symmetrically, seventeens[i] gives
the minimum number of bits we can place assuming all remaining bits are ones (each costing 17 cycles). 
```python
def check_right(i, x):
    j = i + 1
    next_i = 0
    while j < target_ix:
        sum_i = 0
        while j < target_ix and sum_i < x:
            sum_i += states[j]
            j += 1
        if sum_i >= x:
            next_i += 1
    return next_i

for i in range(0, target_ix):
    fives[i] = check_right(i, 5)
    seventeens[i] = check_right(i, 17)
```
We need those tables to create an efficient pruning-tree. Normally if we just used a simple DFS/BFS we would need to check all $2^{256}$ possibilities, which of course is unfeasible. With said arrays we can add one if that completely shifts the dynamic:
```python
def can_reach(i, remaining_bits):
    if i >= target_ix or remaining_bits <= 0:
        return True
    return seventeens[i] <= remaining_bits <= fives[i]

if (pos, i) in memo or not can_reach(i, 256 - pos):
        memo.add((pos, i))
        return None
```
It checks if our position is still possible to achieve cycle number 5720 after 256 bits. If we would need more cycles than we'd get putting only zeros or less cycles putting only ones that means it is impossible to achieve our goal. That means that checked branch is dead and needs to be pruned. You may ask why are we using memo.add. Isn't it different if we had x zeros and y ones compared to m zeros and n ones, even though we ended up in the same position? Well yes, but actually no. Yes, the random password is different, but for us it doesn't matter what precedes our position. If our if disqualifies the branch it means that no matter how our password looks, it won't get the job done. After that we just use a simple DFS/BFS to find the valid password.

**Solver:**
```python
# Step 1: Append states
while r != target:
    states.append(r)
    r = random.random()

# Step 2: precomputes fives and seventeens
def check_right(i, x):
    j = i + 1
    next_i = 0
    while j < target_ix:
        sum_i = 0
        while j < target_ix and sum_i < x:
            sum_i += states[j]
            j += 1
        if sum_i >= x:
            next_i += 1
    return next_i

for i in range(0, target_ix):
    fives[i] = check_right(i, 5)
    seventeens[i] = check_right(i, 17)

# Step 3: DFS on a pruning tree
memo = set()

def consume(i, timeout):
    t = 0
    while t < timeout:
        if i >= len(states):
            return None
        t += states[i]
        i += 1
    return i

def can_reach(i, remaining_bits):
    if i >= target_ix or remaining_bits <= 0:
        return True
    return seventeens[i] <= remaining_bits <= fives[i]

def solve(pos, i, bits):
    if pos == 256:
        if i == target_ix:
            return bits
        return None
    if (pos, i) in memo or not can_reach(i, 256 - pos):
        memo.add((pos, i))
        return None
    for bit in ['0', '1']:
        timeout = 5 if bit == '0' else 17
        new_i = consume(i, timeout)
        if new_i is None:
            continue
        if not can_reach(new_i, 255 - pos):
            continue
        result = solve(pos + 1, new_i, bits + bit)
        if result:
            return result
    memo.add((pos, i))
    return None

# Step 4: recover the flag
password = hex(int(bits, 2))[2:].zfill(64)
io = remote("streams.tamuctf.com", 443, ssl=True, sni="random-password")
io.recvuntil(b"Enter the password in hex: ")
io.sendline(password.encode())
print(io.recvline().decode())
```
